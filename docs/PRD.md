# 커피 주문 앱


## 1. 프로젝트 개요

### 1.1 프로젝트명
커피 주문 앱

### 1.2 프로젝트 목적
사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스택 웹 앱

### 1.3 개발 범위
- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능


## 2. 기술 스택
- 프런트엔드 : HTML, CSS, 리액트, 자바스크립트
- 백엔드 : Node.js, Express
- 데이터베이스 : PostgreSQL


## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음


## 4. 주문하기 화면 상세 사양

### 4.1 레이아웃 구조
- **헤더 영역**
  - 좌측: "COZY" 로고 (다크 그린 배경, 흰색 텍스트, 테두리 포함)
  - 우측: 네비게이션 버튼
    - "주문하기" 버튼 (현재 페이지 표시)
    - "관리자" 버튼

### 4.2 메뉴 아이템 영역
- **메뉴 카드 레이아웃**
  - 메뉴 아이템들을 카드 형태로 가로 배치
  - 각 카드 구성 요소:
    1. **이미지 영역**: 상품 이미지 (플레이스홀더 가능)
    2. **메뉴명**: 예) "아메리카노(ICE)", "아메리카노(HOT)", "카페라떼"
    3. **가격**: 원화 단위로 표시 (예: "4,000원", "5,000원")
    4. **설명**: "간단한 설명..." 형식의 텍스트
    5. **옵션 선택**:
       - "샷 추가 (+500원)" 체크박스
       - "시럽 추가 (+0원)" 체크박스
    6. **담기 버튼**: 회색 버튼, "담기" 텍스트

### 4.3 장바구니 영역
- **위치**: 화면 하단에 고정
- **구성 요소**:
  1. **제목**: "장바구니"
  2. **주문 목록**: 
     - 아이템명, 옵션, 수량 표시 (예: "아메리카노(ICE) (샷 추가) X 1")
     - 각 아이템의 가격 표시 (예: "4,500원")
  3. **총 금액**:
     - "총 금액" 레이블
     - 총 금액 숫자 (굵은 글씨로 강조, 예: "12,500원")
  4. **주문하기 버튼**: 회색 버튼, "주문하기" 텍스트

### 4.4 기능 요구사항
- 메뉴 아이템 목록을 서버에서 조회하여 표시
- 옵션(샷 추가, 시럽 추가) 선택 시 가격에 반영
- "담기" 버튼 클릭 시 장바구니에 추가
- 장바구니에 동일 아이템이 있을 경우 수량 증가
- 장바구니 아이템 목록 실시간 업데이트
- 총 금액 자동 계산
- "주문하기" 버튼 클릭 시 주문 생성 API 호출

### 4.5 UI/UX 디자인
- 배경: 흰색
- 텍스트: 진한 회색
- 로고 박스: 다크 그린 배경, 흰색 텍스트
- 버튼: 회색 배경
- 깔끔하고 미니멀한 디자인


## 5. 관리자 화면 상세 사양

### 5.1 레이아웃 구조
- **헤더 영역**
  - 좌측: "COZY" 로고 (다크 그린 배경, 흰색 텍스트, 테두리 포함)
  - 우측: 네비게이션 버튼
    - "주문하기" 버튼
    - "관리자" 버튼 (현재 페이지 표시 - 테두리 강조)

### 5.2 관리자 대시보드 섹션
- **제목**: "관리자 대시보드"
- **주문 요약 정보**:
  - 형식: "총 주문 {N} / 주문 접수 {N} / 제조 중 {N} / 제조 완료 {N}"
  - 실시간으로 주문 상태별 통계 표시

### 5.3 재고 현황 섹션
- **제목**: "재고 현황"
- **재고 아이템 카드**:
  - 각 메뉴 아이템별로 독립적인 박스 형태
  - 구성 요소:
    1. **아이템명**: 예) "아메리카노 (ICE)", "아메리카노 (HOT)", "카페라떼"
    2. **재고 수량**: "10개" 형식으로 표시
    3. **수량 조절 버튼**:
       - "+" 버튼 (재고 증가)
       - "-" 버튼 (재고 감소)

### 5.4 주문 현황 섹션
- **제목**: "주문 현황"
- **주문 목록**:
  - 각 주문 항목 표시:
    1. **주문 일시**: "7월 31일 13:00" 형식
    2. **주문 내용**: 메뉴명과 수량 (예: "아메리카노(ICE) x 1")
    3. **주문 금액**: "4,000원" 형식
    4. **주문 접수 버튼**: 각 주문 항목 우측에 "주문 접수" 버튼

### 5.5 기능 요구사항
- **주문 통계**:
  - 총 주문 수 실시간 조회
  - 주문 접수 상태 개수 조회
  - 제조 중 상태 개수 조회
  - 제조 완료 상태 개수 조회
- **재고 관리**:
  - 현재 재고 수량 서버에서 조회하여 표시
  - "+" 버튼 클릭 시 재고 증가 API 호출
  - "-" 버튼 클릭 시 재고 감소 API 호출 (0 이하로 내려가지 않도록 제한)
- **주문 관리**:
  - 주문 접수 대기 중인 주문 목록 조회
  - 주문 시간순 정렬
  - "주문 접수" 버튼 클릭 시 해당 주문 상태를 "주문 접수"로 변경
  - 주문 상태 변경 시 대시보드 통계 자동 업데이트

### 5.6 UI/UX 디자인
- 배경: 흰색
- 텍스트: 진한 회색
- 로고 박스: 다크 그린 배경, 흰색 텍스트
- 섹션별로 명확하게 구분된 영역
- 깔끔하고 직관적인 관리자 인터페이스
